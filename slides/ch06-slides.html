Concept 1 / 5

Construction = writing code others can read

Construction is turning the design into working, readable code. Clear names and small functions matter because teammates (and future you) must understand it.

Example
Name it tasks_due_for_reminder(...), not f1(x). The name tells you what it does.
Intro2SE · Ch6
Concept 2 / 5

Version control: why Git exists

Git records the history of your code so a team can work together without overwriting each other, and can undo mistakes. A “commit” is a saved snapshot with a message.

Example
Two teammates edit the same project; Git merges their work and keeps every version, so nothing is lost.
Intro2SE · Ch6
Concept 3 / 5

The GitHub team workflow

Everyone clones the repo, makes commits, and opens pull requests so others can review before merging. Commit history also shows who contributed (it's graded).

Example
Each member commits their own part; the history proves the work was shared, not done by one person.
Intro2SE · Ch6
Concept 4 / 5

AI-assisted coding — then verify

AI can scaffold code fast, but it can be wrong or insecure. You must read it, test it, and be able to explain every line you keep.

Example
AI gives you a reminder function — you run the tests, fix an edge case it missed, and understand it before committing.
Intro2SE · Ch6
Concept 5 / 5

Link code back to the requirement

A good commit message references the requirement it implements — that's the Golden Thread in action.

Example
Commit message: feat: remind open tasks 1 day before due (FR-3)
