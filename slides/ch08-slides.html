Concept 1 / 5

Quality attributes & how to check them

Quality means reliability, usability, maintainability, security, performance. You check it with reviews, tests, and tools — not by hoping.

Example
“Is StudyMate maintainable?” Could a new teammate read the code and add a feature in a day?
Intro2SE · Ch8
Concept 2 / 5

Code review & static checks (lint)

A code review = a teammate reads your code to catch problems early. A linter is a tool that auto-flags style/quality issues. Both are cheap and powerful.

Example
A reviewer spots that your reminder function has no comment and an unclear name; lint flags an unused variable.
Intro2SE · Ch8
Concept 3 / 5

Basic security: the big three for beginners

1) Validate input (never trust user data). 2) Access control (users see only their own data). 3) Never commit secrets (passwords/keys) to the repo.

Example
StudyMate: a student must NOT be able to read another student's tasks — enforce it on the server, not just the screen.
Intro2SE · Ch8
Concept 4 / 5

A vulnerability you can find today

Many beginner apps trust the client. If the server returns data based on an id the user can change, anyone can read others' data.

Example
Bug: GET /tasks?studentId=123 returns whoever you ask for. Fix: use the logged-in user's id, ignore the parameter.
Intro2SE · Ch8
Concept 5 / 5

Privacy & PDPA basics

Personal data (schedules, notes, messages) must be collected with consent and protected. Thailand's PDPA makes this a legal duty, not just good manners.

Example
Don't store more than you need; don't share a student's data without consent.
