Concept 1 / 5

Why test? Verification vs validation

Testing checks the software does the right thing. Verification = “did we build it right?” Validation = “did we build the right thing?” Tests catch bugs before users do.

Example
A reminder that fires on the wrong day passes “it runs” but fails verification — only a test catches it.
Intro2SE · Ch7
Concept 2 / 5

A test = input + expected output

A test gives the code a known input and checks the result equals what you expect. If it doesn't, the test fails and you found a bug.

Example
Input: an open task due 10 Aug, today is 9 Aug. Expected: it appears in the reminder list.
Intro2SE · Ch7
Concept 3 / 5

Unit testing

A unit test checks one small piece (one function) in isolation. They're fast and pinpoint exactly what broke.

Example
Test tasks_due_for_reminder() alone, without the database or UI.
Intro2SE · Ch7
Concept 4 / 5

Acceptance criteria become tests

Each acceptance criterion from your SRS turns into a test. That's how the Golden Thread reaches all the way to testing.

def test_reminds_open_task_1_day_before():
    tasks = [Task(due=date(2026,8,10), status='open')]
    assert tasks_due_for_reminder(tasks, date(2026,8,9)) == tasks

def test_does_not_remind_done_task():
    tasks = [Task(due=date(2026,8,10), status='done')]
    assert tasks_due_for_reminder(tasks, date(2026,8,9)) == []
Intro2SE · Ch7
Concept 5 / 5

Testing AI-generated code

AI code often misses edge cases. Writing tests is how you catch what AI got wrong — and prove your feature works.

Example
AI forgot the “done task” case; your second test fails and reveals it.
